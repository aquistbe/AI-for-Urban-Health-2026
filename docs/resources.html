Installation (General)

All CLI tools require Node.js (version 18 or later). Install via:

# Install Node.js (if not already installed)
# macOS with Homebrew:
brew install node

# Then install the CLI tool:
npm install -g @anthropic-ai/claude-code     # Claude Code
npm install -g @openai/codex                 # Codex CLI

(Google’s terminal agent now ships as part of Antigravity rather than a standalone Gemini CLI.)

Ollama: Local AI Models

What is Ollama?

Ollama lets you run large language models locally on your computer. Your data never leaves your machine (no data is sent to any external server), making it suitable for sensitive data—though the models are generally less capable than cloud-based alternatives.

Installation

Platform Method
macOS brew install ollama or download from ollama.com
Windows Download installer
Linux curl -fsSL https://ollama.com/install.sh \| sh

System Requirements

  • Minimum: 8 GB RAM, 5 GB free disk space
  • Recommended: 16 GB RAM, 10+ GB free disk space
  • For larger models: 32 GB RAM or Apple Silicon Mac with unified memory

Getting Started

# Start Ollama (runs in background)
ollama serve

# Download and run a model (pick one that fits your laptop — see table below)
ollama run qwen3           # Alibaba Qwen3 — strong, good at structured output/tools
ollama run gemma4          # Google Gemma 4 — efficient, laptop-friendly
ollama run qwen3:4b        # smaller size for 8 GB laptops
ollama run llama3.1:8b     # Meta Llama, general purpose

# List downloaded models; see if one is running on GPU or CPU
ollama list
ollama ps

# Remove a model
ollama rm model-name
Note

Open models change fast. The names above are current as of mid-2026 — always check ollama.com/library for the latest tags and sizes. The strongest open-weight model right now, GLM-5.2 (MIT license, ~744B mixture-of-experts), is a datacenter/cloud-class model: run it via Ollama Cloud or a hosted API, not on a laptop.

Which Model Fits Your Laptop?

Rough guide using 4-bit quantization (Q4_K_M, Ollama’s default) — a model needs roughly 0.6 GB of memory per billion parameters, plus headroom for context.

Your laptop Usable memory Models that run well (Q4)
Entry (8 GB RAM, no/integrated GPU) ~5 GB Gemma 4 small, Qwen3 4B, Phi-4-mini, Llama 3.2 3B
Mainstream (16 GB RAM, or 8 GB GPU) ~10 GB Qwen3 8B, Gemma 4, Llama 3.1 8B
Strong (32 GB RAM, or 12–16 GB GPU) ~20 GB Qwen3 14B–32B, Gemma 4 (larger), gpt-oss 20B
Workstation (64 GB+ unified, or 24 GB GPU) 40 GB+ 70B-class or large MoE, heavily quantized
Not a laptop (cloud/server) — GLM-5.2, DeepSeek V4, MiniMax M3, Llama 4 Maverick

On a Mac (Apple Silicon — M1/M2/M3/M4): memory is unified (shared by CPU and GPU) and Ollama uses the GPU (Metal) automatically. Plan to use about 60–70% of total RAM for the model — a 16 GB Mac handles up to ~8–14B comfortably, a 32 GB Mac up to ~32B. Intel Macs are CPU-only; stick to 3–4B models.

On a PC (Windows / Linux): with an NVIDIA GPU, the limit is VRAM — the model should mostly fit in VRAM for good speed (check with nvidia-smi). Roughly: 8 GB VRAM → up to 8B, 12 GB → up to 14B, 24 GB (e.g., RTX 4090) → up to ~32B. Without a dedicated GPU, Ollama runs on CPU + system RAM — fine for 3–8B models, just slower. AMD GPUs work via ROCm on Linux and recent Windows builds.

Browser-Based Alternatives (No Installation Required)

If you cannot install software on your work computer:

  • Google AI Studio: Free, browser-based, runs Gemini models. Reasonable privacy policies for non-sensitive work.
  • HuggingChat: Free, runs open-source models in the browser.
  • Poe: Access to multiple models through a single interface.
Note

Browser-based alternatives still send data to external servers. They are more private than consumer chatbots in some cases, but they are not equivalent to running models locally. For truly sensitive data, use Ollama or discuss options with your IT department.
